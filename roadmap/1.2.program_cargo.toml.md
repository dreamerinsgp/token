Q1: explain the purpose of each section inside program's cargo.toml
```toml
[package]
name = "spl-token"
version = "0.1.0"
description = "Solana Program Library Token"
authors = { workspace = true }
repository = { workspace = true }
license = { workspace = true }
edition = { workspace = true }

[features]
no-entrypoint = []

[dependencies]
arrayref = "0.3.9"
solana-program-error = { workspace = true }
solana-program-pack = { workspace = true }
solana-pubkey = { workspace = true }
solana-program-entrypoint = { workspace = true }
solana-instruction = { workspace = true }
solana-account-info = "3.0.0"
solana-program-memory = "3.0.0"
solana-rent = "3.0.0"
solana-sysvar = { version = "3.0.0", features = ["bincode"] }
solana-msg = "3.0.0"
solana-cpi = "3.0.0"
thiserror = { workspace = true }

[dev-dependencies]
mollusk-svm = "0.6.3"
solana-account = "3.0.0"

[lib]
crate-type = ["cdylib", "lib"]

[package.metadata.solana]
program-id = "TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA"
```

## Answer:

### `[package]` Section
Defines the package metadata:
- **`name`**: Crate name used in imports and compilation
- **`version`**: Semantic version (major.minor.patch)
- **`description`**: Brief description of the program
- **`authors`**, **`repository`**, **`license`**, **`edition`**: Inherited from workspace via `{ workspace = true }` to avoid duplication

### `[features]` Section
Conditional compilation flags:
- **`no-entrypoint`**: When enabled, excludes the program entrypoint. Useful for libraries that provide token functionality without a Solana program entrypoint, allowing reuse in other contexts.

### `[dependencies]` Section
Runtime dependencies required when the program runs on-chain:
- **`arrayref`**: Safe array references without copying
- **`solana-program-error`**: Error types for Solana programs
- **`solana-program-pack`**: Serialization/deserialization utilities
- **`solana-pubkey`**: Public key handling
- **`solana-program-entrypoint`**: Program entrypoint macro (`entrypoint!`)
- **`solana-instruction`**: Instruction parsing and creation
- **`solana-account-info`**: Account information structures
- **`solana-program-memory`**: Memory management utilities
- **`solana-rent`**: Rent calculation and validation
- **`solana-sysvar`**: System variable access (with `bincode` feature for serialization)
- **`solana-msg`**: Message/instruction logging
- **`solana-cpi`**: Cross-Program Invocation (CPI) helpers
- **`thiserror`**: Error handling macros

### `[dev-dependencies]` Section
Dependencies only used during development/testing:
- **`mollusk-svm`**: Solana Virtual Machine for local testing
- **`solana-account`**: Account utilities for testing

### `[lib]` Section
Configures the library output format:
- **`crate-type = ["cdylib", "lib"]`**:
  - **`cdylib`**: Dynamic library format required by Solana runtime (BPF bytecode)
  - **`lib`**: Standard Rust library format for use as a dependency

### `[package.metadata.solana]` Section
Solana-specific metadata:
- **`program-id`**: The on-chain program ID (public key) where this program will be deployed. This is the canonical SPL Token program ID.
